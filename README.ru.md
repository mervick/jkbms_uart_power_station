# Контроллер зарядного устройства JK BMS для зарядной станции

 [english](README.md) | [ру](README.ru.md)

Этот проект предназначен для мониторинга аккумуляторной сборки через **JK BMS** (JiKong) и автоматического управления процессом зарядки с помощью реле.

Модуль работает на **Arduino Pro Micro** (ATmega32U4) и использует аппаратный интерфейс UART для связи с BMS.

Предотвращает микрозапуски, отключая зарядное устройство при достижении **100% SOC** и падении тока заряда ниже безопасного порога насыщения (например, **0.8A**).

## Особенности

- **Hardware UART**: Использует выделенный порт `Serial1` на Pro Micro для стабильной связи с BMS.
- **Умное отключение**: Отслеживает ток и SOC в реальном времени для срабатывания реле и полного отключения зарядного устройства.
- **Защита от самозаряда**: Обнаруживает абсурдно высокое потребление тока во время фазы зарядки (например, если станция ошибочно подключена к собственному инвертору) и немедленно отключает зарядное устройство через реле для предотвращения повреждения оборудования.

## Аппаратная настройка: Arduino Pro Micro

Arduino Pro Micro идеально подходит для этого проекта благодаря своему небольшому размеру и выделенному аппаратному порту (`Serial1`).

### Схема подключения

| Компонент | Пин компонента | Пин Pro Micro | Описание |
| :--- | :--- | :--- | :--- |
| **BMS UART** | **GND** | **GND** | Общая земля |
| **BMS UART** | **TX** | **RX (Pin 0)** | Данные от BMS к Pro Micro |
| **BMS UART** | **RX** | **TX (Pin 1)** | Команды от Pro Micro к BMS *(Требуется делитель напряжения!)* |
| **BMS UART** | **B+ (VCC)** | **НЕ ПОДКЛЮЧАТЬ** | **ОПАСНО: Полное напряжение батареи!** |

| **Модуль реле** | **GND** | **GND** | Общая земля |
| **Вход питания** | **+5V** | **RAW** | Основное питание для Arduino |
| **Вход питания** | **GND** | **GND** | Земля основного питания |

&nbsp;  

> [!WARNING]  
> **⚡ Несовпадение логических уровней (5V vs 3V)**  
> 
> Используемая в этой сборке Arduino Pro Micro работает с логикой 5V, в то время как модуль UART JK BMS работает с логикой 3V. Прямая отправка сигнала 5V с пина `TX` Arduino на пин `RX` BMS может повредить или уничтожить порт UART BMS.  
> Чтобы предотвратить это, в схему на линии `TX` Arduino включен примитивный **делитель напряжения** для понижения напряжения до безопасного уровня 3V для BMS. Пожалуйста, обратитесь к полной принципиальной схеме выше для правильной реализации.

> [!CAUTION]  
> **⚠️ ВЫСОКОЕ НАПРЯЖЕНИЕ на коннекторе UART BMS!**  
> 
> Коммуникационный порт BMS (где расположены пины UART) включает пин, подключенный напрямую к **плюсу батареи (B+)**.  
> На этом пине присутствует **ПОЛНОЕ напряжение батареи** (например, 12V, 24V, 48V+).  
> 
> **НИКОГДА не подключайте его к Arduino!** Оставьте его неподключенным или **полностью отрежьте этот провод**.  
> **Мгновенная смерть Arduino гарантирована!** 💥

## Установка и настройка программного обеспечения

### 1. Настройка Arduino IDE

Чтобы загрузить код в Pro Micro:

1. Откройте **Tools** -> **Board**.
2. Выберите **Arduino Leonardo** или **Arduino Micro** (Pro Micro функционально эквивалентна Leonardo).
3. Выберите правильный **Port**:
* **macOS**: `/dev/cu.usbmodem...`
* **Ubuntu**: `/dev/ttyACM0` (Убедитесь, что у вас есть права: `sudo usermod -a -G dialout $USER`).

### 2. Конфигурация

В файле `.ino` интерфейс `JKBMSInterface` инициализируется с использованием `&Serial1`. Это гарантирует, что ваши отладочные логи идут через USB (`Serial`), в то время как связь с BMS остается на аппаратных пинах 0 и 1 (`Serial1`).

```cpp
Serial1.begin(115200);
// Эта строка в файле .ino использует аппаратные пины Pro Micro (0 и 1)
JKBMSInterface bms(&Serial1);
```

### Рабочий процесс и схема питания

Важно понимать, что Arduino получает питание **только** тогда, когда зарядная станция подключена к внешнему источнику питания (например, к сети переменного тока) для зарядки.

Ниже представлена полная пошаговая логика работы системы:

1. **Включение (Power On):** Вы подключаете станцию к сети (или включаете выделенный тумблер зарядки). Arduino получает питание, и запускается основной скетч. Поскольку питание зарядного устройства проходит через нормально замкнутые — **(NC)** — контакты реле, зарядное устройство включается **мгновенно и автоматически**. Arduino не инициирует зарядку активно; контроллер вмешивается только для того, чтобы отключить её позже.
2. **Сбор данных:** Каждые 10 секунд микроконтроллер запрашивает данные в реальном времени от BMS через `Serial1`.
3. **Защита от самозаряда:** Если система заряжается, но обнаруживается аномально высокое потребление тока (например, станция по ошибке подключена к собственному инвертору), реле срабатывает немедленно, **размыкая цепь и останавливая процесс зарядки**.
4. **Отключение при полном заряде:** Как только достигаются условия отключения (`SOC == 100%` И ток падает `<= 0.8A`), микроконтроллер отправляет сигнал на реле. Реле размыкает цепь, физически отключая зарядное устройство от батареи.
5. **Состояние блокировки (`RELAY_LOCKED`):** Реле остается открытым (зарядное устройство отключено), и Arduino переходит в безопасное состояние `RELAY_LOCKED`. В этот момент сама плата Arduino **остается включенной** (так как получает питание со стороны входа, до размыкания реле). Она продолжает работать, но больше не будет пытаться замкнуть реле. Система безопасно «замораживается», чтобы предотвратить микроциклы включения/выключения зарядного устройства из-за естественной просадки напряжения батареи с течением времени.

**Результат:** Зарядное устройство **не** включится снова автоматически, даже если напряжение батареи позже упадет. Чтобы сбросить систему и начать новый цикл зарядки, необходимо полностью отключить внешнее питание (например, щелкнув выделенным тумблером зарядки или отключив станцию от розетки). Это обесточит Arduino, позволяя начать новый цикл при следующем включении.

## Справочник API (`JKBMSInterface`)

| Метод | Возвращает | Описание |
| --- | --- | --- |
| `getSOC()` | `uint8_t` | Уровень заряда в % (0-100) |
| `getVoltage()` | `float` | Общее напряжение аккумуляторной сборки |
| `getCurrent()` | `float` | Амперы (Положительное = Разряд, Отрицательное = Заряд) |
| `getCellVoltage(i)` | `float` | Напряжение ячейки под индексом `i` |
| `isCharging()` | `bool` | `true`, если ток поступает в батарею |
